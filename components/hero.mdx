## OptimizedVideo Component

Location: `components/optimized-video.tsx`

### Features

- Lazy loading with intelligent preload strategy
- Smooth fade-in transition when loaded
- Gradient placeholder during load
- Autoplay with loop and muted attributes
- Mobile-optimized video file

### Component Code

```tsx
"use client";

import { useState, useRef, useEffect } from 'react';

export default function OptimizedVideo() {
  const [isLoaded, setIsLoaded] = useState(false);
  const videoRef = useRef<HTMLVideoElement>(null);

  useEffect(() => {
    const video = videoRef.current;
    if (!video) return;

    const handleCanPlay = () => {
      setIsLoaded(true);
      video.play().catch(console.error);
    };

    video.addEventListener('canplay', handleCanPlay);
    
    // Intelligent preload
    if (document.readyState === 'complete') {
      video.load();
    } else {
      window.addEventListener('load', () => video.load());
    }

    return () => {
      video.removeEventListener('canplay', handleCanPlay);
    };
  }, []);

  return (
    <>
      {/* Optimized video */}
      <video
        ref={videoRef}
        src="/video/background-mobile.mp4"
        autoPlay
        loop
        muted
        playsInline
        preload="none"
        className={`absolute lg:fixed inset-0 w-full h-full object-cover -z-10 transition-opacity duration-500 ${
          isLoaded ? 'opacity-100' : 'opacity-0'
        }`}
      />
      
      {/* Placeholder that fades out */}
      <div 
        className={`absolute inset-0 bg-linear-to-br from-blue-900 via-blue-800 to-purple-900 -z-10 transition-opacity duration-500 ${
          isLoaded ? 'opacity-0' : 'opacity-100'
        }`}
      />
    </>
  );
}
```

The OptimizedVideo component doesn't accept props currently, but can be extended:

## Performance Optimizations

### Lazy Loading Strategy

```tsx
if (document.readyState === 'complete') {
  video.load();
} else {
  window.addEventListener('load', () => video.load());
}
```

The video only starts loading after the page is fully loaded, prioritizing critical resources.

### Preload Attribute

```html
<video preload="none" />
```

Prevents the browser from automatically downloading video metadata.

### Transition Effects

```css
transition-opacity duration-500
```

Smooth 500ms fade transition between placeholder and video.

### Mobile Positioning

```css
absolute lg:fixed
```

- Mobile: `absolute` positioning within section
- Desktop: `fixed` positioning for parallax effect

## Styling

### Video Overlay

```tsx
<div className="absolute h-full inset-0 bg-black/50 -z-10" />
```

Creates a 50% opacity black overlay to improve text contrast.

### Placeholder Gradient

```css
bg-linear-to-br from-blue-900 via-blue-800 to-purple-900
```

Brand-colored gradient placeholder matches the video aesthetic.

### Responsive Typography

```tsx

```

Text scales from mobile to large desktop screens.

## Accessibility

- Video is muted by default (required for autoplay)
- Overlay ensures text remains readable
- Placeholder provides visual feedback during load
- `playsInline` attribute for iOS compatibility

## Video Requirements

- **Format**: MP4 (H.264 codec recommended)
- **Resolution**: 1920x1080 or lower for mobile
- **File Size**: Optimize for web (target < 5MB)
- **Duration**: Short loops work best (5-15 seconds)
- **Aspect Ratio**: 16:9 recommended

## Browser Support

- Chrome/Edge: Full support
- Firefox: Full support
- Safari: Full support (with `playsInline`)
- Mobile browsers: Optimized with fallback
