# Detailed Installation Guide

This comprehensive guide covers everything you need to install, configure, and deploy the Eufrosine website. Follow these steps for a production-ready setup.

## System Requirements

<Note>
  Ensure your system meets these requirements before starting the installation.
</Note>

### Required Software

<Steps>
  <Step title="Node.js (Version 20 or higher)">
    Eufrosine requires Node.js 20+ for optimal performance and compatibility with React 19.

    **Check your current version:**
    ```bash
    node --version
    # Should output v20.x.x or higher
    ```

    **Install or update Node.js:**
    
    <CodeGroup>
    
    ```bash macOS (Homebrew)
    # Install Node.js via Homebrew
    brew install node@20

    # Or update existing installation
    brew upgrade node
    ```

    ```bash Linux (Ubuntu/Debian)
    # Using NodeSource repository
    curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
    sudo apt-get install -y nodejs

    # Verify installation
    node --version
    npm --version
    ```

    ```bash Windows
    # Download from official website
    # Visit: https://nodejs.org/en/download/

    # Or use Chocolatey
    choco install nodejs-lts

    # Or use Windows Package Manager
    winget install OpenJS.NodeJS.LTS
    ```

    ```bash Using nvm (Recommended)
    # Install nvm first, then:
    nvm install 20
    nvm use 20
    nvm alias default 20
    ```

    </CodeGroup>

    <Warning>
      **Node.js 18 and below are NOT supported** due to React 19 requirements and Next.js 16 dependencies.
    </Warning>
  </Step>

  <Step title="Package Manager">
    Eufrosine supports multiple package managers. Choose one:

### Email System

The contact and complaint forms use **Nodemailer** to send emails via SMTP:

```typescript app/api/contact/route.ts (Simplified)
import nodemailer from "nodemailer";
import { NextResponse } from "next/server";

export async function POST(request: Request) {
  try {
    const { name, email, phone, message } = await request.json();

    // Create transporter
    const transporter = nodemailer.createTransport({
      service: "gmail",
      auth: {
        user: process.env.EMAIL_USER,
        pass: process.env.EMAIL_PASS,
      },
    });

    // Send email
    await transporter.sendMail({
      from: process.env.EMAIL_USER,
      to: process.env.EMAIL_TO,
      subject: `Nuevo mensaje de ${name}`,
      html: `
        <h2>Nuevo mensaje desde el sitio web</h2>
        <p><strong>Nombre:</strong> ${name}</p>
        <p><strong>Email:</strong> ${email}</p>
        <p><strong>Teléfono:</strong> ${phone || "No proporcionado"}</p>
        <p><strong>Mensaje:</strong></p>
        <p>${message}</p>
      `,
    });

    return NextResponse.json(
      { message: "Email sent successfully" },
      { status: 200 }
    );
  } catch (error) {
    console.error("Error sending email:", error);
    return NextResponse.json(
      { error: "Failed to send email" },
      { status: 500 }
    );
  }
}
```

## Deployment

<Tabs>
  <Tab title="Vercel (Recommended)">
    **One-Click Deploy**

    1. Push code to GitHub/GitLab/Bitbucket
    2. Go to [Vercel Dashboard](https://vercel.com/new)
    3. Import your repository
    4. Configure environment variables:
       - Add all variables from `.env.local`
    5. Click "Deploy"

    **Environment Variables in Vercel:**
    - Go to Project Settings → Environment Variables
    - Add each variable from `.env.local`
    - Redeploy after adding variables

    <Check>
      **Automatic Features:**
      - SSL certificate
      - CDN distribution
      - Preview deployments for PRs
      - Automatic builds on push
    </Check>
  </Tab>

  <Tab title="Netlify">
    **Deploy to Netlify**

    1. Build settings:
       - Build command: `npm run build`
       - Publish directory: `.next`
    
    2. Add `netlify.toml` to project root:
       ```toml
       [build]
         command = "npm run build"
         publish = ".next"

       [[plugins]]
         package = "@netlify/plugin-nextjs"
       ```

    3. Set environment variables in Netlify dashboard
    4. Deploy
  </Tab>

  <Tab title="Docker">
    **Containerized Deployment**

    Create `Dockerfile`:
    ```dockerfile
    FROM node:20-alpine AS base

    # Dependencies
    FROM base AS deps
    WORKDIR /app
    COPY package*.json ./
    RUN npm ci

    # Builder
    FROM base AS builder
    WORKDIR /app
    COPY --from=deps /app/node_modules ./node_modules
    COPY . .
    RUN npm run build

    # Runner
    FROM base AS runner
    WORKDIR /app
    ENV NODE_ENV=production
    COPY --from=builder /app/public ./public
    COPY --from=builder /app/.next/standalone ./
    COPY --from=builder /app/.next/static ./.next/static

    EXPOSE 3000
    CMD ["node", "server.js"]
    ```

    Build and run:
    ```bash
    docker build -t eufrosine .
    docker run -p 3000:3000 --env-file .env.local eufrosine
    ```
  </Tab>

  <Tab title="VPS / Traditional Hosting">
    **Deploy to VPS (DigitalOcean, AWS EC2, etc.)**

    1. SSH into server
    2. Install Node.js 20+
    3. Clone repository
    4. Install dependencies:
       ```bash
       npm ci --production
       ```
    5. Create `.env.local` with production values
    6. Build:
       ```bash
       npm run build
       ```
    7. Start with PM2:
       ```bash
       npm install -g pm2
       pm2 start npm --name "eufrosine" -- start
       pm2 save
       pm2 startup
       ```

    **Nginx reverse proxy** (`/etc/nginx/sites-available/eufrosine`):
