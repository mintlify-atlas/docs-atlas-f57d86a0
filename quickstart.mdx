# Quick Start Guide

Get the Eufrosine website up and running on your local machine in just a few minutes. This guide assumes you have basic familiarity with Node.js and Git.

<Note>
  **Prerequisites**: You need Node.js 20+ and npm installed on your machine. Check your versions with `node --version` and `npm --version`.
</Note>

## Quick Setup

<Steps>
  <Step title="Clone the Repository">
    First, clone the Eufrosine repository to your local machine:

    ```bash
    git clone https://github.com/yourusername/proyecto-eufrosine.git
    cd proyecto-eufrosine
    ```

    <Note>
      Replace `yourusername` with the actual repository owner's username.
    </Note>
  </Step>

  <Step title="Install Dependencies">
    Install all required npm packages:

    ```bash
    npm install
    ```

    This will install all dependencies listed in `package.json`, including:
    - Next.js 16.0.10
    - React 19.2.0
    - Framer Motion, GSAP, and Lenis for animations
    - Tailwind CSS 4 for styling
    - Nodemailer for email functionality

    <Warning>
      If you encounter peer dependency warnings, you can safely ignore them or use `npm install --legacy-peer-deps` if needed.
    </Warning>
  </Step>

  <Step title="Configure Environment Variables">
    Create a `.env.local` file in the root directory for email configuration:

    ```bash
    touch .env.local
    ```

    Add your email credentials for the contact form:

    ```bash .env.local
    # Email Configuration (for Nodemailer)
    EMAIL_USER=your-email@gmail.com
    EMAIL_PASS=your-app-password
    EMAIL_TO=aguademesaeufrosineperu@gmail.com
    ```

    <Note>
      **Gmail Users**: Use an [App Password](https://support.google.com/accounts/answer/185833) instead of your regular password for security.
    </Note>

    <Accordion title="Other email providers">
      For other email providers, you'll need to configure the SMTP settings in `/app/api/contact/route.ts`:

      ```typescript
      const transporter = nodemailer.createTransport({
        host: "smtp.yourdomain.com",
        port: 587,
        secure: false,
        auth: {
          user: process.env.EMAIL_USER,
          pass: process.env.EMAIL_PASS,
        },
      });
      ```
    </Accordion>
  </Step>

  <Step title="Start the Development Server">
    Run the development server:

    ```bash
    npm run dev
    ```

    The site will start on [http://localhost:3000](http://localhost:3000)

    <Check>
      You should see the Eufrosine homepage with the hero video and smooth scrolling!
    </Check>
  </Step>

  <Step title="Verify Everything Works">
    Test these key features to ensure proper setup:

    **Navigation**
    - Click on navbar links to test smooth scrolling
    - Test the mobile menu on smaller screens

    **Carousels**
    - Scroll to "Nuestras Presentaciones" section
    - Swipe through the product carousel
    - Check the "Ofertas Especiales" carousel

    **Contact Form** (optional if you configured email)
    - Scroll to the "Contáctanos" section
    - Fill out and submit the form
    - Check for success/error alerts

    <Warning>
      If the contact form doesn't work, ensure your `.env.local` file is configured correctly and the email credentials are valid.
    </Warning>
  </Step>
</Steps>

## Development Commands

Here are the main commands you'll use during development:

<CodeGroup>

```bash Development
# Start development server with hot reload
npm run dev

# Development server runs on http://localhost:3000
# Changes auto-reload thanks to Fast Refresh
```

```bash Build
# Create optimized production build
npm run build

# This generates static HTML and optimized assets
# Build output goes to .next/ directory
```

```bash Production
# Start production server (after build)
npm run start

# Runs the production build on http://localhost:3000
```

</CodeGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Port 3000 is already in use">
    If port 3000 is occupied, you can specify a different port:

    ```bash
    npm run dev -- -p 3001
    ```

    Or kill the process using port 3000:

    ```bash
    # On macOS/Linux
    lsof -ti:3000 | xargs kill -9

    # On Windows
    netstat -ano | findstr :3000
    taskkill /PID <PID> /F
    ```
  </Accordion>

  <Accordion title="Module not found errors">
    If you see "Cannot find module" errors:

    1. Delete `node_modules` and lock file:
       ```bash
       rm -rf node_modules package-lock.json
       ```

    2. Reinstall dependencies:
       ```bash
       npm install
       ```

    3. Restart the dev server
  </Accordion>

  <Accordion title="Smooth scrolling not working">
    The Lenis smooth scroll requires JavaScript to be enabled. Check:

    1. Browser console for JavaScript errors
    2. That the `<LenisSmoothScroll />` component is rendered in `layout.tsx`
    3. Clear browser cache and hard reload (Ctrl+Shift+R)
  </Accordion>

  <Accordion title="Images not loading">
    Images are served from the `/public` directory. Ensure:

    1. Images exist in `/public/img/` directory
    2. File paths in code match actual file names (case-sensitive)
    3. Image files are not corrupted

    Example:
    ```tsx
    // Correct: /public/img/logo.png
    <img src="/img/logo.png" alt="Logo" />

    // Wrong: missing /public prefix
    <img src="public/img/logo.png" alt="Logo" />
    ```
  </Accordion>

  <Accordion title="Contact form not sending emails">
    Email issues are usually related to environment variables:

    1. Verify `.env.local` exists and has correct credentials
    2. For Gmail, use an [App Password](https://support.google.com/accounts/answer/185833)
    3. Check the browser console and terminal for error messages
    4. Test with a simple curl request to `/api/contact`

    ```bash
    curl -X POST http://localhost:3000/api/contact \
      -H "Content-Type: application/json" \
      -d '{"name":"Test","email":"test@example.com","message":"Test message"}'
    ```
  </Accordion>
</AccordionGroup>
